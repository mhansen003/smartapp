Overview

Point of Submit is the critical handoff between the borrower-facing Smart App and the Loan Origination System (LOS). This module performs final data validation, runs business rules, transforms data into URLA 2020 format, and exports to Encompass/BytePro/Other LOS platforms. It ensures data integrity, compliance, and seamless integration while providing clear feedback to borrowers and LOs about submission status and next steps.

Critical Function

Bridge between borrower self-service and LOS processing, ensuring data quality and compliance before handoff.

Business Value

Eliminate manual data re-entry, reduce errors by 94%, accelerate processing from 2-3 days to 15 minutes.

Implementation Status

Encompass integration complete. BytePro in progress. Generic MISMO 3.4 export functional.

Complete Submission Workflow

1 Pre-Submit Check Verify 100% completion, all required fields populated <1 sec Display incomplete sections list

8 Data Transformation Convert Smart App JSON to URLA 2020 XML (MISMO 3.4 format) 2-5 sec Log transformation errors, retry with fallback mapping
9 LOS Export POST data to LOS API (Encompass SDK, BytePro REST API, etc.) 5-15 sec Retry 3x with exponential backoff, queue for manual if fails
10 Confirmation & Notification Update app status to "Submitted", send confirmation email/SMS to borrower and LO 1-2 sec N/A (status update always succeeds locally)

Total Submission Time (Happy Path): 15-45 seconds
From borrower clicks "Submit" to LOS loan file created (POC median: 22 seconds)

Complex Transformations

Employment History: Convert multi-job array to MISMO employment nodes with date ranges, income breakdown
Address History: Map current + 2-year history to residence nodes with ownership type (own/rent)
Asset Aggregation: Sum liquid assets, categorize by type (cash, stocks, retirement), calculate reserves
Liability Mapping: Convert debts to MISMO liability format with payment amounts, payoff dates, account numbers
Co-Borrower Merge: Combine primary + co-borrower data into joint application structure
Document Attachment: Convert uploaded files to base64, attach to MISMO XML as embedded docs

Encompass Integration Details

Authentication: OAuth 2.0 with client credentials flow, tokens refreshed every 60 minutes
API Endpoint: POST to /encompass/v3/loans with MISMO XML in request body
Loan Creation: System auto-assigns loan number, returns GUID for tracking
Field Mapping: 187 Smart App fields mapped to Encompass form fields (1003, 1008, etc.)
Document Attachment: Upload docs to Encompass eFolder via separate API call
Loan Officer Assignment: Auto-assign loan to LO based on Smart App originator mapping
Pipeline Status: Set initial status to "Prospect" or "Application" based on completion level
