Overview

Prove's identity verification platform uses phone number intelligence to instantly verify borrower identity without requiring document uploads or knowledge-based authentication (KBA). By analyzing phone possession, ownership history, and device signals, Prove delivers real-time identity confidence scores while detecting fraud attempts and synthetic identities.

Integration Purpose

Verify borrower identity in seconds using phone number as the primary authentication factor, reducing friction and abandonment.

Business Value

Increase application completion rates by 35%, reduce fraud losses by 90%, and accelerate verification from days to seconds.

Implementation Status

Integrated in Smart App POC with 98.2% success rate. Production ready pending final fraud rule configuration.

Verification Workflow

Step Action User Experience System Process Duration
1 Phone Number Entry Borrower enters mobile phone number Validate format and carrier <1 sec
2 Phone Intelligence Check Seamless background process Query Prove API for phone ownership, tenure, risk signals 1-2 sec
3 Identity Match Enter name, DOB, last 4 SSN Match provided PII against phone owner records <1 sec
4 Possession Challenge Receive SMS with 6-digit code Send OTP to verified phone number 5-10 sec
5 Code Entry Enter 6-digit code from SMS Validate OTP within 10-minute window 5-15 sec
6 Trust Score Calculation See "Verified" status instantly Calculate composite trust score (0-1000) <1 sec
7 Result Display Proceed to next application step Store verification result, trust score, and audit trail <1 sec
Total Verification Time: 15-30 seconds
Compared to 2-3 days for traditional document-based verification

Security & Compliance

All API calls over TLS 1.3 with certificate pinning
API keys stored in secure vault (AWS Secrets Manager)
PII encrypted at rest with AES-256-GCM
Audit logging of all verification attempts with timestamp, IP, device fingerprint
TCPA compliance for SMS delivery (consent capture required)
GLBA compliance for data retention (7 years for completed applications)
Rate limiting: 5 verification attempts per phone per hour
SOC 2 Type II certified infrastructure

Error Scenarios & Handling

Error Type Cause User Message Resolution
PHONE_NOT_FOUND Phone number not in Prove database "We couldn't verify this phone number. Please try a different number or contact us." Allow alternate phone or manual verification
PHONE_MISMATCH Phone owner name doesn't match provided identity "The information you provided doesn't match our records. Please verify your details." Allow correction or escalate to LO review
OTP_EXPIRED Verification code entered after 10-minute window "This code has expired. We'll send you a new one." Automatically resend new OTP
OTP_INVALID Incorrect code entered "That code isn't correct. Please try again. (2 attempts remaining)" Allow retry up to configured limit
RATE_LIMIT_EXCEEDED Too many verification attempts "Too many attempts. Please wait 1 hour or contact your loan officer." Enforce cooldown period or LO override
SIM_SWAP_DETECTED Phone number recently ported to new SIM "For security, we need additional verification. Your loan officer will contact you." Flag for fraud review, notify LO
VOIP_DETECTED Phone is VoIP/Google Voice, not mobile carrier "Please use your mobile phone number for verification." Request different phone or document verification
SMS_DELIVERY_FAILED Carrier rejected SMS delivery "We couldn't send a text to this number. Would you like a voice call instead?" Offer voice call delivery fallback
