Overview

Every borrower's journey through Smart App is unique. The Borrower Scenario Logic defines all possible pathways, conditional displays, and decision points that create a personalized, relevant experience. This includes property type variations, loan purpose differences, employment scenarios, and co-borrower workflows.

Key Scenario Examples

Scenario 1: First-Time Homebuyer Purchase

Step Question/Screen Logic/Condition Next Step
1 What are you looking to do? User selects "Purchase a home" Skip to property search
2 Is this your first home? User selects "Yes" Show first-time buyer programs
3 Have you found a property? If No: Show home search resources Continue to pre-qualification
4 Property type User selects property type Conditional questions based on type
5 Down payment amount Calculate LTV, show PMI if <20% Continue to income section

Scenario 2: Cash-Out Refinance

Step Question/Screen Logic/Condition Next Step
1 What are you looking to do? User selects "Cash-out Refinance" Current mortgage questions
2 Current property address Auto-fill from Prove if available Verify with AVM
3 Current mortgage balance User enters amount Calculate available equity
4 Desired cash-out amount Validate against LTV limits Show estimated new payment
5 Purpose of cash-out Required disclosure Continue to income verification

Scenario 3: Self-Employed Borrower

Step Question/Screen Logic/Condition Alternative Path
1 Employment status User selects "Self-Employed" Skip TRUV, show manual entry
2 Business name and type Required for self-employed Collect business details
3 Years in business If <2 years, flag for review May need additional docs
4 Monthly income Show info: "We'll verify with tax returns" Set expectation for docs
5 Business expenses Optional field for more accurate calculation Calculate qualifying income

Implementation Considerations

All conditional logic must be testable and documented
Default values should guide borrower to most common path
Progressive disclosure keeps initial screens simple
Back button must correctly handle conditional sections
Analytics should track which paths borrowers take
Edge cases require LO review flag, not blocking errors
Scenario matrix must be maintained with each feature addition
