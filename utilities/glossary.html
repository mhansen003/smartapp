How to Use This Glossary

Each term includes its category, definition, context within the project, and related terms. Use the alphabet navigation to jump to specific letters, filter by category, or search for specific terms.

A

APR (Annual Percentage Rate) Business
The annual cost of a loan expressed as a percentage, including interest and fees. APR provides borrowers with a more complete picture of loan costs than interest rate alone.
Smart App 2.0 Context:
Displayed to borrowers during the pre-qualification phase and calculated based on loan amount, term, and lender fees. Must comply with TRID disclosure requirements.
Related Terms:
TRID Clear to Close
API (Application Programming Interface) Technical
A set of protocols and tools that allows different software applications to communicate with each other. APIs define the methods and data formats applications can use to request and exchange information.
Smart App 2.0 Context:
Smart App 2.0 integrates with multiple third-party APIs including Plaid (VOA), TRUV (VOIE), and Prove (identity verification). All integrations use RESTful APIs with JWT authentication.
Example:
When a borrower connects their bank account, Smart App calls the Plaid API to retrieve account balances and transaction history for asset verification.
Related Terms:
REST JWT OAuth 2.0 Plaid VOA
A/B Testing Product/UX
A method of comparing two versions of a webpage, feature, or user flow to determine which performs better. Users are randomly assigned to version A or B, and metrics are compared to identify the winning variant.
Smart App 2.0 Context:
Used to test progressive disclosure patterns, form layouts, and AI assistant interactions. Critical for optimizing conversion rates and completion times.
Related Terms:
Conversion Funnel Progressive Disclosure MVP
Application Takeover Project-Specific
A feature allowing Loan Officers to intervene and assist borrowers who have stalled or abandoned their application. LOs can review progress, complete remaining sections, or guide borrowers through difficult areas.
Smart App 2.0 Context:
LO Dashboard provides real-time visibility into application status. When a borrower is stuck, LOs receive alerts and can take over the application to provide personalized assistance.
Related Terms:
LO Dashboard LO Drop-off Rate
AUS (Automated Underwriting System) Business
Software that automatically evaluates mortgage loan applications based on borrower credit, income, assets, and property information. Returns a recommendation (approve, refer, or decline) within seconds.
Smart App 2.0 Context:
Smart App integrates with Desktop Underwriter (DU) and other AUS systems to provide instant preliminary decisions. Results inform borrowers of their likelihood of approval.
Example:
After submitting their application, a borrower receives an instant "Approved with Conditions" result from DU, listing required documentation.
Related Terms:
DU Conditional Approval Point of Submit
AWS (Amazon Web Services) Technical
Amazon's cloud computing platform providing on-demand computing resources including servers, storage, databases, networking, and more. Used for hosting scalable applications.
Smart App 2.0 Context:
Smart App 2.0 infrastructure runs on AWS using ECS for container orchestration, RDS for PostgreSQL databases, S3 for document storage, and CloudFront CDN for global content delivery.
Related Terms:
ECS S3 PostgreSQL CDN

B

Bolt Prototype Project-Specific
A high-fidelity interactive prototype of Smart App 2.0 built using Bolt.new, demonstrating the complete borrower journey including Prove integration, progressive disclosure, and co-borrower collaboration.
Smart App 2.0 Context:
The Bolt prototype serves as a proof-of-concept for stakeholder review and user testing. It validated key UX patterns and integration approaches before full development.
Related Terms:
POC Prototype MVP
Borrower Persona Product/UX
A fictional representation of a target user based on research data, including demographics, goals, pain points, behaviors, and motivations. Used to guide design decisions and ensure solutions meet user needs.
Smart App 2.0 Context:
Research identified three primary borrower personas: First-Time Homebuyers (tech-savvy, needs guidance), Refinancers (experienced, values speed), and Move-Up Buyers (busy professionals, needs mobile access).
Related Terms:
User Journey UAT
Borrower Self-Service Project-Specific
The core capability of Smart App 2.0 allowing borrowers to complete mortgage applications independently without constant LO assistance. Includes automated data collection, instant verification, and real-time guidance.
Smart App 2.0 Context:
Designed to reduce LO workload while improving borrower experience. Features progressive disclosure, AI assistant support, and smart integrations to make the process intuitive and efficient.
Related Terms:
Progressive Disclosure AI Assistant LO Dashboard

C

CDN (Content Delivery Network) Technical
A geographically distributed network of servers that cache and deliver web content from locations closest to users, reducing load times and improving performance.
Smart App 2.0 Context:
CloudFront CDN delivers static assets (images, CSS, JavaScript) globally, ensuring fast page loads regardless of borrower location.
Related Terms:
AWS S3
CI/CD (Continuous Integration/Continuous Deployment) Technical
A development practice where code changes are automatically tested and deployed to production. CI ensures code integrates properly; CD automates the release process.
Smart App 2.0 Context:
GitHub Actions pipeline automatically runs tests, builds containers, and deploys to AWS ECS when code is merged to main branch. Enables rapid iteration and reliable releases.
Related Terms:
ECS Microservices
Clear to Close Business
The final approval stage where all conditions have been satisfied, documentation is complete, and the loan is ready for closing. Borrowers can schedule their closing appointment.
Smart App 2.0 Context:
Smart App tracks application progress from submission through clear to close, providing borrowers with real-time status updates and next steps.
Related Terms:
Conditional Approval Point of Submit TRID
Co-Borrower Flow Project-Specific
A collaborative workflow where multiple borrowers (spouses, partners) can work on the same application simultaneously or asynchronously, each completing their own sections.
Smart App 2.0 Context:
Primary borrower can invite co-borrowers via email/SMS. Real-time synchronization using WebSockets ensures both see live updates. Research shows 28% higher completion rates with co-borrower collaboration.
Related Terms:
WebSocket LO Dashboard
Conditional Approval Business
A preliminary loan approval subject to specific conditions being met, such as providing additional documentation, explanation letters, or asset verification.
Smart App 2.0 Context:
After AUS evaluation, borrowers receive conditional approval with a clear list of outstanding requirements. Smart App guides them through fulfilling each condition.
Related Terms:
AUS DU Clear to Close
Conversion Funnel Product/UX
The journey users take from initial awareness to completion of a goal (application submission). Each stage represents a step where users can convert or drop off.
Smart App 2.0 Context:
Key funnel stages: Landing ‚Üí Registration ‚Üí Identity Verification ‚Üí Application Start ‚Üí Completion ‚Üí Submit. Analytics track drop-off rates at each stage to identify friction points.
Related Terms:
Drop-off Rate Friction Point A/B Testing

D

DTI (Debt-to-Income Ratio) Business
A percentage calculated by dividing total monthly debt payments by gross monthly income. Lenders use DTI to assess a borrower's ability to manage monthly payments and repay debt.
Smart App 2.0 Context:
Smart App calculates DTI automatically based on income verification (TRUV) and declared debts. Most conventional loans require DTI below 43-50%.
Example:
Borrower has $5,000 monthly income and $1,500 in monthly debts (credit cards, car loan). DTI = $1,500 / $5,000 = 30%.
Related Terms:
LTV VOIE TRUV
DU (Desktop Underwriter) Business
Fannie Mae's automated underwriting system that evaluates loan applications and provides risk assessment recommendations. The most widely used AUS in the mortgage industry.
Smart App 2.0 Context:
Smart App submits complete application data to DU at point of submit, receiving instant feedback on loan eligibility and required documentation.
Related Terms:
AUS Conditional Approval Point of Submit
Drop-off Rate Product/UX
The percentage of users who abandon a process or page without completing the desired action. High drop-off rates indicate friction points or usability issues.
Smart App 2.0 Context:
Analytics track drop-off at each application stage. Progressive disclosure reduced drop-off by 134% by breaking long forms into digestible steps.
Related Terms:
Conversion Funnel Friction Point Progressive Disclosure

E

ECS (Elastic Container Service) Technical
Amazon's container orchestration service for running and managing Docker containers. ECS handles deployment, scaling, and management of containerized applications.
Smart App 2.0 Context:
Smart App microservices run as Docker containers orchestrated by ECS. Auto-scaling ensures the application handles variable load during peak times.
Related Terms:
AWS Microservices CI/CD
Epic Product/UX
A large body of work that can be broken down into smaller user stories. Epics typically represent major features or initiatives spanning multiple sprints.
Smart App 2.0 Context:
Examples include "VOA Integration," "Progressive Disclosure Implementation," and "AI Assistant Features." Each epic contains 5-15 user stories.
Related Terms:
User Story Sprint Story Points

F

FICO Score Business
A credit score ranging from 300-850 that represents a borrower's creditworthiness. Calculated based on payment history, credit utilization, length of credit history, new credit, and credit mix.
Smart App 2.0 Context:
Smart App performs soft credit pulls to retrieve FICO scores without impacting credit. Scores inform loan eligibility and interest rate estimates.
Related Terms:
Soft Credit Pull APR
Friction Point Product/UX
Any element in the user experience that causes frustration, confusion, or abandonment. Friction points increase cognitive load and reduce conversion rates.
Smart App 2.0 Context:
Research identified key friction points: long forms (solved with progressive disclosure), complex financial terms (solved with AI assistant), and unclear progress indicators (solved with visual progress bars).
Related Terms:
Drop-off Rate Progressive Disclosure User Journey

G

GraphQL Technical
A query language for APIs that allows clients to request exactly the data they need. More flexible than REST, enabling clients to specify the structure of responses.
Smart App 2.0 Context:
Considered for LO Dashboard real-time queries where complex data relationships and selective field retrieval improve performance over REST.
Related Terms:
REST API

J

JWT (JSON Web Token) Technical
A compact, URL-safe token format for securely transmitting information between parties. JWTs contain claims and are digitally signed to verify authenticity.
Smart App 2.0 Context:
Used for borrower authentication and API authorization. After OTP verification, borrowers receive a JWT that authenticates all subsequent API requests.
Related Terms:
OAuth 2.0 OTP API

L

LO (Loan Officer) Business
A licensed professional who helps borrowers through the mortgage application process, provides loan options, gathers documentation, and guides them to closing.
Smart App 2.0 Context:
LOs use the LO Dashboard to monitor borrower progress, configure Smart App settings, and intervene when borrowers need assistance. Smart App reduces LO workload by 42% through automation.
Related Terms:
LO Dashboard Application Takeover LOS
LO Dashboard Project-Specific
A real-time monitoring interface for Loan Officers showing all active applications, borrower progress, completion percentages, and alert notifications for stalled applications.
Smart App 2.0 Context:
Uses WebSockets for live updates. LOs see which sections borrowers are completing, can configure Smart App settings, and can take over applications to provide assistance.
Related Terms:
LO WebSocket Application Takeover
LOS (Loan Origination System) Business
Enterprise software that manages the entire mortgage loan process from application to closing, including data management, compliance, underwriting, and document generation.
Smart App 2.0 Context:
Smart App integrates with existing LOS platforms (Encompass, Calyx) at point of submit, transferring all borrower data and documents into the LOS for final processing.
Related Terms:
Point of Submit URLA LO
LTV (Loan-to-Value Ratio) Business
The ratio of the loan amount to the appraised value of the property, expressed as a percentage. Lower LTV ratios indicate less risk for lenders.
Smart App 2.0 Context:
Smart App calculates LTV based on purchase price or appraised value and loan amount. Most lenders require LTV below 80% to avoid PMI (Private Mortgage Insurance).
Example:
Home value: $400,000. Loan amount: $320,000. LTV = $320,000 / $400,000 = 80%.
Related Terms:
DTI VOA

M

MFA (Multi-Factor Authentication) Technical
A security system requiring two or more verification factors to grant access. Combines something you know (password), something you have (phone), and/or something you are (biometrics).
Smart App 2.0 Context:
Borrowers authenticate using OTP sent to their verified phone number, providing multi-factor security without passwords.
Related Terms:
OTP Prove
Microservices Technical
An architectural approach where an application is built as a collection of small, independent services that communicate via APIs. Each service handles a specific business function.
Smart App 2.0 Context:
Smart App uses microservices architecture with separate services for authentication, integrations (Plaid, TRUV, Prove), document management, and application processing. Enables independent scaling and deployment.
Related Terms:
Monolith API ECS
Mockup Product/UX
A high-fidelity static visual representation of a design showing colors, typography, images, and exact layout. More detailed than wireframes but not interactive.
Smart App 2.0 Context:
Figma mockups were created for all Smart App screens before development, ensuring visual consistency and stakeholder alignment on design direction.
Related Terms:
Wireframe Prototype
Monolith Technical
A traditional software architecture where all components are tightly coupled and deployed as a single unit. Simpler to develop initially but harder to scale and maintain as complexity grows.
Smart App 2.0 Context:
Smart App 2.0 migrated from a monolithic architecture to microservices to enable independent scaling of integrations and better fault isolation.
Related Terms:
Microservices
MVP (Minimum Viable Product) Product/UX
The simplest version of a product that can be released to validate core assumptions and gather user feedback. Contains only essential features needed to solve the primary problem.
Smart App 2.0 Context:
Phase 1 MVP includes core application flow, Prove integration, progressive disclosure, and basic LO dashboard. Advanced features like AI assistant and full co-borrower collaboration come in later phases.
Related Terms:
POC Sprint

O

OAuth 2.0 Technical
An authorization framework that enables applications to obtain limited access to user accounts on another service. Allows secure delegated access without sharing passwords.
Smart App 2.0 Context:
Plaid, TRUV, and Prove integrations use OAuth 2.0 for secure third-party authorization. Borrowers grant Smart App permission to access their data without exposing credentials.
Related Terms:
JWT API Plaid VOA
OTP (One-Time Password) Project-Specific
A temporary password valid for a single login session. Typically sent via SMS or email and expires after a short period (5-10 minutes).
Smart App 2.0 Context:
Primary authentication method for borrowers. After entering phone number, borrowers receive a 6-digit code via SMS to verify identity and access their application.
Related Terms:
MFA Prove JWT

P

Plaid VOA Project-Specific
Plaid's Verification of Assets integration allowing borrowers to connect bank accounts and instantly verify assets without uploading bank statements.
Smart App 2.0 Context:
Borrowers authenticate with their bank through Plaid's secure interface. Smart App retrieves account balances, transaction history, and asset verification reports for underwriting.
Related Terms:
VOA API OAuth 2.0
POC (Proof of Concept) Product/UX
A small-scale demonstration to verify that a concept or theory has practical potential. Used to test feasibility before committing to full development.
Smart App 2.0 Context:
The Bolt prototype served as POC for progressive disclosure, Prove integration, and real-time collaboration. Validated technical feasibility and user acceptance.
Related Terms:
MVP Prototype Bolt Prototype
Point of Submit Project-Specific
The final validation and submission stage where borrower data is validated, consent is obtained, and the complete application package is transferred to the LOS and AUS.
Smart App 2.0 Context:
Smart App performs comprehensive validation, ensures all required fields are complete, obtains e-signatures, runs soft credit pull, and submits to DU for preliminary decision.
Related Terms:
LOS DU URLA
PostgreSQL Technical
An open-source relational database management system known for reliability, feature robustness, and standards compliance. Supports complex queries and transactions.
Smart App 2.0 Context:
Primary database for Smart App 2.0 with 42 tables managing borrower data, applications, integrations, LO configurations, and audit trails. Hosted on AWS RDS.
Related Terms:
Redis AWS
Progressive Disclosure Project-Specific
A UX pattern that reveals information and form fields gradually, showing only what's relevant at each step. Reduces cognitive load and prevents users from feeling overwhelmed.
Smart App 2.0 Context:
Core UX strategy breaking the URLA application into digestible sections. Research showed 134% increase in completion rates. Shows borrowers 3-5 questions at a time instead of 50+ fields on one screen.
Example:
Rather than showing all employment questions upfront, Smart App first asks "Are you currently employed?" then progressively reveals employer details, income, and employment history based on answers.
Related Terms:
Friction Point Drop-off Rate User Journey
Prove Project-Specific
Phone-centric identity verification service that validates borrower identity using phone number, device intelligence, and carrier data. Provides instant identity confirmation and data prefill.
Smart App 2.0 Context:
Primary identity verification method. Prove validates borrower phone ownership, retrieves identity data (name, address, DOB), and prefills form fields reducing data entry by 60%.
Related Terms:
OTP MFA API
Prototype Product/UX
An interactive simulation of a product allowing users to experience and test functionality before full development. Can range from low-fidelity (clickable wireframes) to high-fidelity (code-based).
Smart App 2.0 Context:
Bolt prototype demonstrates the complete borrower experience with realistic interactions, API integrations, and data flow. Used for stakeholder demos and user testing.
Related Terms:
Bolt Prototype Mockup Wireframe
Pull-through Rate Business
The percentage of loan applications that successfully close. A key metric for lenders measuring the efficiency of their loan origination process.
Smart App 2.0 Context:
Smart App aims to improve pull-through rates by reducing abandonment, improving data quality, and providing better borrower experience. Target: increase from 65% to 82%.
Related Terms:
Conversion Funnel Drop-off Rate

R

RBAC (Role-Based Access Control) Technical
A security approach where access permissions are assigned based on user roles rather than individual users. Simplifies permission management and improves security.
Smart App 2.0 Context:
Different roles have different permissions: Borrowers (view/edit own application), LOs (view all applications, configure settings), Admins (full system access, user management).
Related Terms:
Multi-tenancy JWT
Redis Technical
An in-memory data store used for caching, session management, and real-time data. Extremely fast due to storing data in RAM rather than disk.
Smart App 2.0 Context:
Used for session storage, caching integration responses (Plaid, TRUV), and managing WebSocket connections for real-time LO Dashboard updates.
Related Terms:
PostgreSQL WebSocket
REST (Representational State Transfer) Technical
An architectural style for designing networked applications using standard HTTP methods (GET, POST, PUT, DELETE). REST APIs are stateless and use resource-based URLs.
Smart App 2.0 Context:
All third-party integrations (Plaid, TRUV, Prove) use RESTful APIs. Smart App also exposes REST endpoints for LO Dashboard and mobile applications.
Related Terms:
API GraphQL

S

S3 (Simple Storage Service) Technical
Amazon's object storage service for storing and retrieving any amount of data. Highly durable, scalable, and secure storage for files, images, backups, and more.
Smart App 2.0 Context:
All uploaded documents (pay stubs, bank statements, tax returns) are stored in encrypted S3 buckets with lifecycle policies for compliance and retention.
Related Terms:
AWS CDN
Smart App 2.0 Project-Specific
The reimagined digital mortgage application platform designed to provide borrowers with an intuitive, self-service experience while reducing LO workload through automation and smart integrations.
Project Context:
Complete rebuild of existing Smart App with modern UX patterns (progressive disclosure), third-party integrations (Plaid, TRUV, Prove), real-time collaboration, AI assistance, and comprehensive LO tools.
Related Terms:
Progressive Disclosure LO Dashboard AI Assistant
Soft Credit Pull Project-Specific
A credit inquiry that does not impact the borrower's credit score. Allows lenders to check creditworthiness without the "hard pull" required for final approval.
Smart App 2.0 Context:
Performed at point of submit to retrieve FICO scores and credit history for AUS evaluation. Borrowers see estimated rates based on soft pull results.
Related Terms:
FICO Score Point of Submit AUS
Sprint Product/UX
A time-boxed development period (typically 1-2 weeks) where a team commits to completing specific user stories. Core component of Agile development methodology.
Smart App 2.0 Context:
Smart App 2.0 development uses 2-week sprints with planning, daily standups, demo, and retrospective. Each sprint delivers working software increments.
Related Terms:
Epic User Story Velocity
SSE (Server-Sent Events) Technical
A server push technology enabling servers to send real-time updates to clients over HTTP. Simpler than WebSockets for one-way communication from server to client.
Smart App 2.0 Context:
Considered for LO Dashboard notifications but WebSockets were chosen for bidirectional communication needs.
Related Terms:
WebSocket LO Dashboard
Story Points Product/UX
A relative measure of effort required to complete a user story, considering complexity, uncertainty, and volume of work. Used for sprint planning and capacity management.
Smart App 2.0 Context:
Team uses Fibonacci sequence (1, 2, 3, 5, 8, 13) for story pointing. Team velocity averages 42 points per sprint.
Related Terms:
User Story Sprint Velocity

T

TRID (TILA-RESPA Integrated Disclosure) Business
Federal regulations requiring lenders to provide borrowers with specific disclosures about mortgage costs and terms at defined times during the loan process. Includes Loan Estimate and Closing Disclosure forms.
Smart App 2.0 Context:
Smart App generates TRID-compliant disclosures automatically based on borrower data and loan terms. Ensures timing compliance for 3-day review periods.
Related Terms:
APR Clear to Close
TRUV VOIE Project-Specific
TRUV's Verification of Income and Employment integration providing instant employment and income verification through payroll system connections, eliminating need for pay stubs and W2s.
Smart App 2.0 Context:
Borrowers connect their employer's payroll system through TRUV. Smart App receives employment history, income amounts, payment frequency, and YTD earnings directly from source.
Related Terms:
VOIE DTI API

U

UAT (User Acceptance Testing) Product/UX
Testing performed by end users to verify the system meets business requirements and works as expected in real-world scenarios. Final validation before production release.

Phase 3 includes comprehensive UAT with 10 LOs and 25 borrowers testing all workflows, integrations, and edge cases before go-live.

MVP POC
URLA (Uniform Residential Loan Application) Business
The standardized mortgage application form (Form 1003) used by all lenders. URLA 2020 version includes updated fields for employment, income, assets, and demographics.

Smart App collects all URLA 2020 fields through progressive disclosure. Complete gap analysis ensures 100% compliance with URLA requirements before LOS transfer.

LOS Point of Submit

Borrower Persona Friction Point Conversion Funnel

VOA (Verification of Assets) Business
The process of confirming borrower assets (bank accounts, investments, retirement accounts) for mortgage qualification. Verifies down payment funds and reserves.
Smart App 2.0 Context:
Automated through Plaid integration, allowing instant asset verification without manual document upload. Reduces processing time from days to seconds.
Related Terms:
Plaid VOA VOIE LTV
VOIE (Verification of Income and Employment) Business
The process of confirming borrower employment status and income for mortgage qualification. Verifies job stability and ability to repay the loan.
Smart App 2.0 Context:
Automated through TRUV integration connecting directly to payroll systems. Eliminates need for pay stubs, W2s, and employer verification calls.
Related Terms:
TRUV VOIE VOA DTI

W

WebSocket Technical
A communication protocol providing full-duplex, bidirectional channels over a single TCP connection. Enables real-time data exchange between client and server.

Powers real-time features including LO Dashboard live updates, co-borrower collaboration synchronization, and instant notification delivery.

LO Dashboard Co-Borrower Flow SSE
